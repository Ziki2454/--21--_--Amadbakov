# ИС_Склад

## Информационная система: Склад

> **Предметная область:** [Товары / Категории / Поставщики / Заказы]
> **Разработчик:** Группа [21ИС-9-24], [Бозоров Ораз, Амадбеков Зикрие,Гуламадзода Акмал,Абдулов Мурад]

## 1. Описание проекта
Данный проект представляет собой прототип складской системы для удобного учёта товаров, управления поставщиками, обработки заказов и контроля остатков на складе.  
Система позволяет структурировать хранение данных и быстро получать аналитическую информацию через SQL-запросы.

## 2. Техническое задание (ТЗ) вкратце
* **Цель:** Цифровизация складского учета: от ручных журналов к реляционной базе данных на SQLite.
* **Основные функции:**
   1. Регистрация поставщиков и поступлений товара.
   2. Учет заказанных единиц продукции.
   3. Контроль текущих остатков на складе.
   4. Формирование отчетов по отгрузкам за период.
* **Требования к БД:** Минимум 4 связанные таблицы, соблюдение целостности данных.

## 3. Структура базы данных (ER-схема)
Проект базируется на следующих сущностях:
1. **[Товары]** — справочник товаров (название, цена, количество на складе).
2. **[Категории]** — справочник категорий товаров (например, "Электроника", "Инструменты").
3. **[Поставщики]** — компании-поставщики (название, контакт, адрес).
4. **[Заказы]** — фиксирует отгрузку товара (связь: товар, поставщик, количество, дата).

*Скриншот ER-диаграммы находится в папке `/diagrams`.*

## 4. Реализованные SQL-запросы
В рамках проекта подготовлены сценарии для:
1. Подсчета текущих остатков: вывод всех товаров с количеством на складе.
2. Выборки с JOIN: Вывод заказов с названиями товаров и поставщиков вместо ID.
3. Фильтрации: Поиск заказов за заданный период.
4. Аналитики: Топ-3 самых заказываемых товаров за месяц.

## Структура репозитория
1. /data — файл базы данных warehouse.db
2. /sql — скрипты создания таблиц (schema.sql) и тестовые данные (seeds.sql)
3. /diagrams — графическая схема базы данных (er_diagram.png)
4. queries.sql — файл с примерами сложных запросов (отчеты)

## Как запустить (Инструкция)
1. Установите DB Browser for SQLite или используйте консоль sqlite3.
2. Склонируйте репозиторий: git clone [ссылка на ваш репозиторий].
3. Откройте файл /data/warehouse.db или выполните последовательно скрипты из /sql/ для создания БД с нуля.
4. Запустите запросы из файла queries.sql для проверки работоспособности.
