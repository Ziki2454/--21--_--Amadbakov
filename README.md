1. Описание проекта
Данный проект представляет собой прототип информационной системы (ИС) для автоматизации складского учета. Система позволяет структурировать хранение данных о товарах, поставщиках, категориях и заказах, а также быстро получать аналитическую информацию через SQL-запросы и консольное приложение на Python.
2. Техническое задание (ТЗ) вкратце
•	Цель: Перейти от бумажного учета к автоматизированной базе данных на SQLite с удобным консольным интерфейсом.
•	Основные функции:
1.	Регистрация новых записей (товары, заказы).
2.	Поиск данных по заданным критериям (товары с низким остатком).
3.	Формирование итоговых отчетов (список товаров, детали заказов).
4.	Обновление остатков на складе.
5.	Изменение статусов заказов.
•	Требования к БД: Минимум 4 связанные таблицы, соблюдение целостности данных (внешние ключи).
3. Структура базы данных (ER-схема)
Проект базируется на следующих сущностях:
1.	Categories (Категории) — хранит данные о категориях товаров (название, описание).
2.	Suppliers (Поставщики) — хранит данные о поставщиках (название, контактное лицо, телефон, email).
3.	Products (Товары) — основная сущность, связывает категории и поставщиков (название, цена, остаток).
4.	Orders (Заказы) — хранит данные о заказах (дата, статус).
5.	Order_items (Позиции заказа) — связывает товары и заказы (количество, цена на момент заказа).
Связи между таблицами:
•	Одна категория может содержать много товаров (1:M)
•	Один поставщик может поставлять много товаров (1:M)
•	Один заказ может содержать много позиций (1:M)
•	Один товар может быть во многих заказах (M:M через order_items)
Скриншот ER-диаграммы находится в папке /docs.
4. Реализованные SQL-запросы
В рамках проекта подготовлены сценарии для:
•	Подсчета общего количества товаров на складе.
•	Выборки данных с использованием JOIN (объединение таблиц товаров, категорий и поставщиков).
•	Фильтрации данных по условию (товары с остатком менее 5 штук).
•	Детального отчета по заказам с расчетом итоговой суммы.
